# 이대로 실험해도 될까?

중학교 과학 탐구 계획서를 **실험 전에 스스로 점검**하는 웹 도구입니다.
학생이 계획서를 쓰는 동안, 변인이 잴 수 있는 값인지 · 과정이 구체적인지 · 위험한 것은 없는지를 실시간으로 짚어 줍니다.

**답을 대신 써 주지 않습니다.** 모든 피드백은 학생이 스스로 고칠 수 있도록 질문 형태로 제시됩니다.

## 바로 쓰기

https://samsunk25-hue.github.io/science-inquiry-checker/

파일 하나짜리 정적 페이지입니다. `index.html`을 내려받아 더블클릭해도 똑같이 동작합니다.

## 설계 원칙

**AI를 쓰지 않습니다.** 규칙 기반 검사만으로 동작합니다.

- Gemini API 등 상용 LLM API는 약관상 18세 미만 대상 서비스에 사용할 수 없습니다
- 따라서 학생 동선에서 AI 호출을 완전히 배제했습니다
- 그 결과 API 키·서버·비용·대기열·연령 제한 문제가 모두 사라졌습니다

**네트워크 요청이 0건입니다.** 외부 스크립트, 웹폰트, 이미지, 분석 도구를 일절 쓰지 않습니다. 학교 방화벽 뒤에서도, 인터넷이 끊겨도 동작합니다.

**개인정보를 수집하지 않습니다.** 이름·학번 입력란이 없습니다. 작성 내용은 브라우저의 localStorage에만 임시 저장되며 어디로도 전송되지 않습니다.

## 점검 항목

2022 개정 교육과정의 탐구 요소를 네 갈래로 나누어 약 25가지를 검사합니다.

| 갈래 | 예 |
|---|---|
| 가설 | 조건(`-면`/`-수록`)과 결과의 방향이 드러나는가, 조작·종속 변인이 문장에 포함되는가 |
| 변인 통제 | **재거나 정할 수 있는 값인가**, 조작 변인이 하나인가, 통제 변인이 충분한가, 변인끼리 겹치지 않는가 |
| 탐구 설계 | 번호 매긴 절차인가, 수치와 단위가 있는가, 측정 도구가 명시됐는가, 반복 측정과 대조군이 있는가 |
| 현실성·안전 | 위험 물질·화기·전기 등 8개 위험군 감지, 주의 사항과 뒷정리 계획 |

**변인 판정이 이 도구의 핵심입니다.** "식물의 건강", "잘 자란 정도"처럼 사람마다 다르게 보는 표현을 잡아내고, "운동하는 시간"과 "잠자는 시간"처럼 단위 낱말만 겹치는 경우를 서로 다른 변인으로 구별합니다.

## 결과 내보내기

계획서 전문과 점검 결과를 하나의 파일로 저장합니다. `.html` · `.doc`(한글·워드) · `.txt` 중에서 고를 수 있고, 인쇄로 PDF 저장도 됩니다.

## 규칙 고치기

판정에 쓰는 어휘 목록은 `index.html` 안의 `DIM`(값의 종류) · `VAGUE`(모호한 말) · `SAFETY`(위험 물질) 등에 모여 있습니다. 수업에서 자주 나오는 표현을 여기에 추가하면 됩니다.

고친 뒤에는 반드시 테스트를 돌려 주세요.

```
node _test-rules.js
```

`index.html`에서 정규식을 직접 읽어와 실제 학생 답안 예시로 검사합니다. 규칙을 고쳐도 이미 해결한 오탐이 되살아나지 않도록 막아 줍니다.
